## 📝 报告类型

### 1️⃣ 日报（Daily Report）

**频率**: 每日  
**时间**: 21:00 前完成  
**长度**: 2000-5000 字  
**重点**: 具体工作内容、即时问题、当日反思

**必填内容**:
- [ ] 今日概要（核心目标 + 主要成果）
- [ ] 完成的工作（至少 1 项）
- [ ] 经验教训（至少 1 条）
- [ ] 待执行事项

**使用场景**:
- 记录每日工作细节
- 追踪问题和解决过程
- 次日工作参考

---

### 2️⃣ 周报（Weekly Report）

**频率**: 每周日  
**时间**: 21:00 前完成  
**长度**: 3000-8000 字  
**重点**: 周度汇总、趋势分析、周计划

**必填内容**:
- [ ] 本周概览（目标完成度）
- [ ] 每日摘要
- [ ] 重点项目进展
- [ ] 深度反思
- [ ] 下周计划

**使用场景**:
- 周度复盘
- 项目进度追踪
- 周计划制定

---

### 3️⃣ 月报（Monthly Report）

**频率**: 每月最后一天  
**时间**: 21:00 前完成  
**长度**: 5000-15000 字  
**重点**: 战略复盘、月度规划、能力成长

**必填内容**:
- [ ] 月度概览（核心成就）
- [ ] 周度摘要
- [ ] 投资表现分析
- [ ] 重大决策复盘
- [ ] 下月规划

**使用场景**:
- 月度战略复盘
- 投资表现分析
- 月度/季度规划

## 📖 使用指南

### 创建新报告

1. **复制模板**
```bash
# 日报
cp reports/templates/日报模板.md reports/daily/$(date +%Y-%m-%d) 工作日报.md

# 周报
cp reports/templates/周报模板.md reports/weekly/$(date +%Y-W%V) 周报.md

# 月报
cp reports/templates/月报模板.md reports/monthly/$(date +%Y-%m) 月报.md
```

2. **填写内容**
   - 替换 `{{VARIABLE}}` 占位符
   - 删除不适用章节
   - 添加额外内容

3. **保存归档**
   - 使用 UTF-8 编码
   - 文件名包含日期
   - 提交 git（如适用）

---

### 查找历史报告

**按日期查找**:
```bash
# 查找特定日期的日报
ls reports/daily/ | grep 2026-03-28

# 查找特定周次的周报
ls reports/weekly/ | grep 2026-W14

# 查找特定月份的月报
ls reports/monthly/ | grep 2026-03
```

**按内容查找**:
```bash
# 搜索关键词
grep -r "多因子" reports/

# 搜索特定主题
grep -r "Bug 修复" reports/daily/
```

## 💡 最佳实践

### ✅ 应该做的

1. **及时性**
   - 日报：当日 21:00 前
   - 周报：周日 21:00 前
   - 月报：月末 21:00 前

2. **具体性**
   - 用数据说话
   - 引用具体文件/代码
   - 记录时间戳

3. **真实性**
   - 不回避问题
   - 坦诚反思错误
   - 记录真实想法

4. **可追溯**
   - 文件路径完整
   - 代码片段准确
   - 决策依据清晰

5. **可行动**
   - 待执行事项明确
   - 责任人和时间
   - 优先级标注

---

### ❌ 应该避免的

1. **流水账**
   - ❌ "今天做了很多工作"
   - ✅ "完成多因子 v5.0 策略开发（22KB，7 大因子）"

2. **模糊描述**
   - ❌ "修复了一些 Bug"
   - ✅ "修复动量因子日期匹配 Bug（20:00-20:22）"

3. **只报喜不报忧**
   - ❌ "一切顺利"
   - ✅ "发现 2 个严重 Bug，已修复"

4. **缺少反思**
   - ❌ "完成了任务"
   - ✅ "完成任务，但发现日期处理有缺陷，已改进"

5. **待执行不明确**
   - ❌ "继续优化"
   - ✅ "周日 10:00 前完成回测验证（multi_factor_backtest_v5.py）"
