### 4️⃣ 晚间重大 Bug 修复（20:00-20:57 完成）

#### Bug #1: 动量/波动因子全部 0.500 ❌

**问题发现** (19:31):
```
动量因子：min=0.500, max=0.500, 分差=0.000 ❌
波动因子：min=0.500, max=0.500, 分差=0.000 ❌
```

**根因定位** (20:00):
- 脚本使用当前日期 `20260328`（周六，非交易日）
- 历史数据最新日期是 `20260327`（周五）
- 日期不匹配 → 所有 ETF 被跳过 → 返回默认值 0.500

**修复方案** (20:15):
```python
# 修复前
trade_date = datetime.now().strftime('%Y%m%d')  # 20260328 ❌

# 修复后
all_dates = set()
for data in historical_data.values():
    all_dates.update(data.keys())
trade_date = max(all_dates)  # 20260327 ✅
```

**修复成果** (20:22):
| 指标 | 修复前 | 修复后 | 改善 |
|------|--------|--------|------|
| 动量因子分差 | 0.000 | 0.803 | ✅ 有效 |
| 波动因子分差 | 0.000 | 0.983 | ✅ 有效 |
| 动量均值 | 0.500 | 0.166 | ✅ 真实 |
| 波动均值 | 0.500 | 0.418 | ✅ 真实 |

## 🧠 经验教训

### 1️⃣ 日期处理陷阱 ⚠️

**教训**: 永远不要假设当前日期是交易日

**问题**: 
```python
trade_date = datetime.now().strftime('%Y%m%d')  # 周六/周日/节假日 ❌
```

**正确做法**:
```python
# 从历史数据中获取最新交易日期
all_dates = set()
for data in historical_data.values():
    all_dates.update(data.keys())
trade_date = max(all_dates) if all_dates else None  # ✅
```

**应用范围**: 所有涉及交易日期的场景

**后续措施**: 
- ✅ 已更新 `screen_all_etf_v5_repair.py`
- ✅ 在相关函数中添加日期验证逻辑

### 2️⃣ API 速率限制管理 ⚠️

**教训**: Tushare 速率限制严格（200 次/分钟），需要分批处理

**问题**:
```python
# 一次性获取 608 只 ETF 名称 → 触发限制 ❌
etf_info = get_etf_basic(etf_codes)
```

**正确做法**:
```python
# 分批获取，每批 50 只，间隔 1.5 秒
batch_size = 50
for i in range(0, len(etf_codes), batch_size):
    batch = etf_codes[i:i+batch_size]
    batch_info = get_etf_basic(batch)
    time.sleep(1.5)  # ✅
```

**应用范围**: 所有批量 API 调用场景

**后续措施**:
- ✅ 已更新 `fix_etf_names.py` 支持分批处理
- ✅ 在 `tushare_finance_data.py` 中添加速率限制处理
