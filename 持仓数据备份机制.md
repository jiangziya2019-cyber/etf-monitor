# 持仓数据备份与恢复机制

## 📦 备份策略

### 三重备份保护

| 备份类型 | 位置 | 触发时机 | 保留策略 |
|---------|------|---------|---------|
| **自动备份** | `holdings_backup/holdings_YYYYMMDD_HHMMSS.json` | 每次更新持仓数据时 | 保留最近 30 个 |
| **最新备份** | `holdings_backup/holdings_latest.json` | 每次更新持仓数据时 | 永久保留（总是覆盖） |
| **定时备份** | `holdings_backup/` | 每日 16:10（cron） | 保留最近 30 个 |

## 🔧 使用方法

### 1️⃣ 手动备份

```bash
# 创建备份
python3 /home/admin/openclaw/workspace/backup_holdings.py backup

# 查看备份列表
python3 /home/admin/openclaw/workspace/backup_holdings.py list

# 验证当前文件和备份
python3 /home/admin/openclaw/workspace/backup_holdings.py verify
```

### 2️⃣ 安全更新持仓数据

```bash
# 从 JSON 文件更新
python3 /home/admin/openclaw/workspace/safe_update_holdings.py new_holdings.json

# 从 stdin 更新
cat new_holdings.json | python3 /home/admin/openclaw/workspace/safe_update_holdings.py
```

**安全更新流程：**
1. 自动备份现有数据
2. 写入新数据到临时文件
3. 验证新数据格式
4. 原子替换（避免损坏）
5. 创建新备份

### 3️⃣ 从备份恢复

```bash
# 从最新备份恢复
python3 /home/admin/openclaw/workspace/backup_holdings.py restore

# 从指定备份恢复
python3 /home/admin/openclaw/workspace/backup_holdings.py restore holdings_20260327_223054.json
```

## ⏰ 自动备份任务

### Cron 配置

| 任务 | 时间 | 说明 |
|------|------|------|
| **每日 16:10 备份** | 交易日 16:10 | 盘后自动备份持仓数据 |

### cron 表达式
```
10 16 * * 1-5  # 周一至周五 16:10
```

## 🛡️ 数据安全保障

### 更新流程保护

```
原始文件 → 备份 → 临时文件 → 验证 → 原子替换 → 新备份
   ↓                                              ↓
   └────────────── 可恢复 ────────────────────────┘
```

### 验证项目

每次更新前自动验证：
- ✅ JSON 格式有效性
- ✅ 必要字段存在（last_updated, total_market_value, etfs）
- ✅ ETF 列表非空
- ✅ 数值类型正确

### 原子操作

使用 `rename()` 原子替换，避免写入过程中文件损坏。
