# 每日工作总结 Skill v1.0

## 描述
标准化的每日工作总结流程，用于复盘当日工作、记录重要事项、规划明日计划。

## 触发条件

### 主动触发
当用户提到以下关键词时启动：
- "每日总结"、"工作总结"、"今日总结"
- "开始总结"、"进行总结"、"写总结"
- "盘后复盘"、"每日复盘"

### 定时触发
- **时间**: 每日 23:00（如果老板未主动提出）
- **方式**: 主动询问老板是否需要总结
- **超时处理**: 如长时间无回复，自动生成并存储，次日提醒确认

## 总结流程

### 步骤 1: 生成工作总结
**内容结构**:
1. ✅ 今日完成工作（任务清单）
2. 🎯 核心成果（关键数据/指标）
3. ✅ 数据准确性检查（无假设/编造）
4. ⚠️ 问题与改进（教训记录）
5. 📅 明日计划（P0/P1/P2 优先级）
6. ❓ 需要老板确认（待决策事项）

### 步骤 2: 发送给老板确认
- 发送完整总结
- 等待老板确认/修改
- 记录反馈意见

### 步骤 3: 发送到飞书（同步记录）
- 使用飞书 App Bot API
- 私聊发送给老板
- 作为正式记录存档

### 步骤 4: 更新记忆系统
- 追加到 `memory/YYYY-MM-DD.md`
- 确保记忆完整
- 标记为已确认/待确认

### 步骤 5: 准备明日工作
- 确认明日优先事项
- 准备相关模块/数据
- 设置闹钟/提醒

### 步骤 6: 结束今日工作
- 回复 **HEARTBEAT_OK**
- 准备明日执行

## 定时提醒机制

### 23:00 主动询问
**触发条件**: 23:00 老板未提出总结
**询问方式**:
```
老板，今日工作总结还未进行，是否需要现在开始？

如需要，我将：
1. 生成今日工作总结
2. 发送给您确认
3. 同步到飞书
4. 更新记忆系统

如不需要，我将：
- 自动生成并存储
- 明日提醒您确认
```

### 超时处理
**条件**: 23:30 仍无回复
**处理**:
1. 自动生成总结
2. 存储到记忆系统
3. 标记为"待确认"
4. 次日 08:00 提醒老板确认

### 次日提醒
**时间**: 次日 08:00
**内容**:
```
老板早安！

昨晚已自动生成工作总结（待确认）：
- 存储位置：memory/YYYY-MM-DD.md
- 状态：待确认

请在方便时确认或修改。
```

## 注意事项

1. **数据准确性**
   - ✅ 所有数据必须查证
   - ❌ 不能假设/编造
   - ✅ 标注数据来源

2. **教训记录**
   - ✅ 必须记录今日教训
   - ✅ 写入记忆系统
   - ✅ 避免重复犯错

3. **优先级管理**
   - P0: 必须完成
   - P1: 尽量完成
   - P2: 可延后

4. **时间控制**
   - 23:00 主动询问
   - 23:30 自动生成
   - 次日 08:00 提醒

5. **记忆存储**
   - ✅ 使用 `memory/YYYY-MM-DD.md`
   - ❌ 不创建 timestamped  variant
   - ✅ 追加模式（不覆盖）
