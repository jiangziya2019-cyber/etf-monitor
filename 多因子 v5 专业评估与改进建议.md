#### 2. 因子标准化方法不当 ⭐⭐⭐⭐⭐

**当前问题**:
```python
# 当前使用 Min-Max 归一化
score = (value - min) / (max - min)  # 归一化到 0-1

# 问题:
# 1. 对极端值敏感
# 2. 分布不均匀
# 3. 不同因子不可比
```

**专业做法**:
```python
# 使用 Z-Score 标准化
def zscore_normalize(factor_values):
    """Z-Score 标准化"""
    mean = np.mean(factor_values)
    std = np.std(factor_values)
    if std < 1e-6:
        return np.zeros_like(factor_values)
    
    z_scores = (factor_values - mean) / std
    
    # Winsorization（缩尾处理）
    z_scores = np.clip(z_scores, -3, 3)  # 限制在±3σ
    
    # 转换为 0-1 评分
    scores = 0.5 + z_scores / 6  # 映射到 0-1
    
    return np.clip(scores, 0, 1)
```

**影响**:
- 当前方法对极端值敏感
- 因子评分分布不均匀
- 可能高估/低估某些 ETF

**建议**: **立即改用 Z-Score 标准化 + Winsorization**

#### 3. 行业中性化缺失 ⭐⭐⭐⭐⭐

**问题**:
```
当前策略可能产生巨大的行业暴露:
- 如果科技行业估值低 + 动量强 → 可能配置 50%+ 科技
- 行业利空时，策略大幅回撤

示例:
2021 年科技股崩盘，行业集中策略回撤>40%
```

**解决方案**:
```python
def industry_neutralize(scores, industry_exposures):
    """
    行业中性化处理
    
    Args:
        scores: 原始评分
        industry_exposures: 行业暴露矩阵
    """
    from sklearn.linear_model import LinearRegression
    
    # 对每个行业回归，取残差
    model = LinearRegression()
    model.fit(industry_exposures, scores)
    
    # 预测值（行业影响）
    industry_effect = model.predict(industry_exposures)
    
    # 残差（去除行业影响后的评分）
    neutralized_scores = scores - industry_effect
    
    return neutralized_scores

# 使用示例
etf_industries = {
    '510300': '宽基',
    '515790': '科技',
    '512480': '科技',
    '510880': '金融',
    ...
}

neutralized_scores = industry_neutralize(raw_scores, etf_industries)
```

**建议**: **必须加入行业中性化，控制行业暴露<15%**

#### 4. 流动性因子缺失 ⭐⭐⭐⭐

**问题**:
```
当前策略未考虑流动性风险:
- 可能选到日均成交<1000 万的 ETF
- 调仓时冲击成本巨大
- 极端情况下无法卖出

示例:
2020 年 3 月流动性危机，低流动性 ETF 跌幅额外 -20%
```

**解决方案**:
```python
def calculate_liquidity_score(etf_codes, historical_data):
    """
    计算流动性评分
    
    因子:
    - 日均成交额 (50%)
    - 换手率 (30%)
    - 买卖价差 (20%)
    """
    liquidity_scores = {}
    
    for code in etf_codes:
        # 日均成交额（20 日）
        avg_turnover = np.mean(historical_data[code]['turnover'][-20:])
        
        # 换手率
        turnover_rate = historical_data[code]['turnover_rate'][-1]
        
        # 评分
        score_turnover = min(avg_turnover / 100000000, 1) * 0.5  # 1 亿为满分
        score_rate = min(turnover_rate / 5, 1) * 0.3  # 5% 为满分
        
        liquidity_score = score_turnover + score_rate
        
        liquidity_scores[code] = liquidity_score
    
    return liquidity_scores

# 整合到综合评分
composite_score = (
    0.25 * valuation +
    0.25 * momentum +
    0.15 * volatility +    # 降低到 15%
    0.10 * technical +
    0.15 * fund_flow +
    0.10 * liquidity       # 新增流动性 10%
)
```

**建议**: **加入流动性因子（10% 权重），设置流动性门槛（日均成交>5000 万）**

#### 5. ETF 特定因子缺失 ⭐⭐⭐⭐

**问题**:
```
ETF 与股票不同，有特有因子:
- 溢价率/折价率（QDII 尤其重要）
- 跟踪误差
- 管理费/托管费
- 规模（避免清盘风险）

当前策略完全忽略这些 ETF 特有因子
```

**解决方案**:
```python
def calculate_etf_specific_factors(etf_codes):
    """
    计算 ETF 特有因子
    
    因子:
    - 溢价率 (40%)
    - 跟踪误差 (30%)
    - 费率 (20%)
    - 规模 (10%)
    """
    etf_scores = {}
    
    for code in etf_codes:
        # 1. 溢价率（从实时数据计算）
        premium_rate = get_premium_rate(code)
        score_premium = 1 - abs(premium_rate) / 0.05  # 5% 以内满分
        
        # 2. 跟踪误差（年化）
        tracking_error = get_tracking_error(code)
        score_te = 1 - min(tracking_error / 0.05, 1)  # 5% 以内满分
        
        # 3. 费率（年费率）
        expense_ratio = get_expense_ratio(code)
        score_fee = 1 - min(expense_ratio / 0.015, 1)  # 1.5% 以内满分
        
        # 4. 规模（亿元）
        fund_size = get_fund_size(code)
        score_size = min(fund_size / 50, 1)  # 50 亿满分
        
        # 综合评分
        etf_specific_score = (
            0.4 * score_premium +
            0.3 * score_te +
            0.2 * score_fee +
            0.1 * score_size
        )
        
        etf_scores[code] = etf_specific_score
    
    return etf_scores
```

**建议**: **加入 ETF 特有因子（10% 权重），特别是溢价率监控**

```python
# 指数衰减加权
def exponential_decay_weights(returns, half_life):
    """指数衰减加权"""
    decay = np.exp(-np.arange(len(returns)) * np.log(2) / half_life)
    return np.average(returns, weights=decay)

# 不同因子使用不同半衰期
```

#### 7. 因子 IC 分析缺失 ⭐⭐⭐⭐

**问题**:
```
没有计算每个因子的信息系数（IC）:
- 无法评估因子有效性
- 无法动态调整因子权重
- 可能使用失效因子

IC < 0.02 的因子应该剔除
```

**解决方案**:
```python
def calculate_factor_ic(factor_scores, future_returns):
    """
    计算因子 IC（信息系数）
    
    IC = Corr(factor_score, future_return)
    """
    from scipy.stats import pearsonr
    
    ic, p_value = pearsonr(factor_scores, future_returns)
    
    return {
        'ic': ic,
        'p_value': p_value,
        't_stat': ic / np.sqrt((1 - ic**2) / (len(factor_scores) - 2)),
        'valid': p_value < 0.05 and abs(ic) > 0.02
    }

# 定期评估因子 IC
factor_ic = calculate_factor_ic(current_scores, future_20d_returns)

if not factor_ic['valid']:
    # 因子失效，降低权重或剔除
    FACTOR_WEIGHTS[factor_name] *= 0.5
```

**建议**: **加入因子 IC 监控，动态调整失效因子**

#### 8. 风险模型缺失 ⭐⭐⭐⭐

**问题**:

```
- 无 VaR/CVaR 计算
- 无压力测试
- 无极端情况应对

2020 年 3 月、2015 年股灾等极端情况未考虑
```

**解决方案**:
```python
def calculate_portfolio_risk(portfolio_returns):
    """
    计算组合风险指标
    """
    # VaR (95%)
    var_95 = np.percentile(portfolio_returns, 5)
    
    # CVaR (预期亏损)
    cvar_95 = np.mean(portfolio_returns[portfolio_returns <= var_95])
    
    # 最大回撤
    cumulative = (1 + np.array(portfolio_returns)).cumprod()
    max_dd = np.max((np.maximum.accumulate(cumulative) - cumulative) / np.maximum.accumulate(cumulative))
    
    return {
        'var_95': var_95,
        'cvar_95': cvar_95,
        'max_drawdown': max_dd
    }

# 风险限制
if risk_metrics['max_drawdown'] > 0.15:
    # 降低仓位
    target_positions *= 0.7
```

**建议**: **加入 VaR/CVaR 风控，设置最大回撤限制**
