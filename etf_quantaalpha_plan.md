# ETF-QuantaAlpha 融合分析方案

## 一、方案概述

**目标**：将 QuantaAlpha 的轨迹级进化方法适配到 ETF 投资分析场景，提升：
1. 观察池筛选的量化科学性
2. 止损/止盈触发的智能判断
3. 行业前景评估的因子化
4. ETF 轮动信号的自动挖掘

**核心原则**：
- 保持现有三层漏斗架构不变
- 增量引入量化因子方法，不颠覆现有流程
- 所有因子必须有经济逻辑解释（不黑箱）
- 数据延迟容忍（15 分钟级别，非高频）

## 二、QuantaAlpha 方法论适配

### 2.1 核心概念映射

| QuantaAlpha 概念 | 原义（股票因子） | ETF 适配版本 |
|-----------------|-----------------|-------------|
| **Alpha 因子** | 个股超额收益预测信号 | ETF 轮动信号/行业景气度信号 |
| **Mining Trajectory** | 假设→因子构建→回测的完整流程 | 假设→ETF 筛选规则→历史验证 |
| **Mutation** | 定位失败步骤针对性修正 | 调整筛选阈值/增加过滤条件 |
| **Crossover** | 重组高收益父代片段 | 融合多个有效筛选规则 |
| **Semantic Consistency** | 假设/表达式/代码一致 | 逻辑/规则/执行一致 |
| **Complexity Control** | 符号长度≤200,特征≤4 | 规则≤5 条，数据源≤3 个 |
| **Redundancy Control** | AST 匹配去重 | 规则相似度过滤 |

### 2.2 轨迹定义（ETF 场景）

```
ETF 筛选轨迹 τ = (s₀, a₀, s₁, a₁, ..., sₙ)

s₀: 初始上下文
  - 市场状态（牛市/熊市/震荡）
  - 行业轮动阶段
  - 用户种子规则（可选）

aᵢ: 代理行动
  - 生成筛选假设
  - 构建筛选规则
  - 执行历史回测
  - 评估结果

sₙ: 终端状态
  - 筛选出的 ETF 列表
  - 回测绩效指标
  - 规则可信度评分
```

## 三、四大核心模块设计

### 3.1 模块 A：多样化规划初始化

**目的**：生成多个互补的 ETF 筛选方向，避免单一逻辑过拟合

**初始化方向池**（示例）：

| 方向 | 逻辑 | 数据源 |
|------|------|--------|
| **动量轮动** | 强者恒强，追涨杀跌 | 20/60 日收益率 |
| **均值回归** | 超跌反弹，估值修复 | PE/PB 分位数 |
| **资金流向** | 主力进场，量价配合 | 成交量/资金流 |
| **波动率目标** | 低波稳健，高波弹性 | ATR/历史波动率 |
| **行业景气** | 产业链逻辑，政策驱动 | 宏观/行业数据 |
| **跨境套利** | 溢价折价，汇率对冲 | IOPV/汇率 |

**输出**：6-10 个初始筛选假设，覆盖不同市场逻辑

---

### 3.2 模块 B：因子实现与约束门控

**步骤**：

1. **假设形式化**
   ```
   自然语言："低估值 + 高动量的 ETF 表现更好"
   ↓
   结构化描述：
   - 估值条件：PE 分位数 < 30%
   - 动量条件：20 日收益 > 0
   - 组合方式：AND
   - 排序规则：按动量降序
   ```

2. **规则编译**
   ```python
   def screen_etf(etf_data):
       # 估值过滤
       mask_pe = etf_data['pe_percentile'] < 30
       # 动量过滤
       mask_mom = etf_data['return_20d'] > 0
       # 组合
       selected = etf_data[mask_pe & mask_mom]
       # 排序
       return selected.sort_values('return_20d', ascending=False)
   ```

3. **约束门控**（关键！）
   - **语义一致性检查**：规则是否忠实于原始假设？
   - **复杂度检查**：规则数量≤5？数据源≤3？
   - **冗余检查**：与现有规则相似度<80%？

---

### 3.3 模块 C：轨迹级进化

**Mutation（变异）流程**：

```
输入：低绩效轨迹 τ（如观察池筛选连续 2 周跑输基准）

1. 自反思定位失败步骤
   - 是假设问题？（逻辑本身错误）
   - 是参数问题？（阈值太紧/太松）
   - 是数据问题？（数据源延迟/噪声）

2. 针对性修正
   - 假设问题 → 生成新假设
   - 参数问题 → 调整阈值（如 PE 分位 30%→40%）
   - 数据问题 → 切换数据源/增加平滑

3. 保持其他步骤不变，重新执行后续流程

输出：修正后的轨迹 τ_child
```

**Crossover（交叉）流程**：

```
输入：2 个高绩效父代轨迹 τ₁, τ₂

τ₁: 动量规则（20 日收益排序）→ 绩效好
τ₂: 估值规则（PE 分位过滤）→ 回撤低

交叉操作：
τ_child = τ₁.动量排序 + τ₂.估值过滤

输出：融合规则（低估值 + 高动量）
```

**进化调度**：
- 每周日执行一次完整进化迭代
- 每月评估是否需要大幅变异
- 保留 Top 3 规则进入"规则池"

## 四、与现有体系对接

### 4.1 观察池筛选增强

**现有**：人工经验 + 简单量化指标

**增强后**：
```
每月初执行：
1. 运行 QuantaAlpha 进化流程
2. 输出 Top 15-20 只 ETF 观察池
3. 附带每只 ETF 的入选理由（因子贡献分解）
4. 人工审核确认（保持人在回路）
```

### 4.2 止损/止盈智能分析

**现有**：固定阈值 + 行业前景评分（人工）

**增强后**：
```
触发器监控时：
1. 检测止损/止盈条件
2. 调用进化后的行业景气因子
3. 输出智能建议：
   - ✅ 建议执行（因子确认趋势恶化）
   - 🟡 部分执行（因子信号分歧）
   - ⚠️ 暂缓执行（因子确认暂时波动）
```

**示例**：
```
黄金 9999 止损触发 (-10%)
├─ 行业景气因子：3.2/5（中性）
├─ 资金流因子：-0.3（小幅流出）
├─ 波动率因子：高（市场恐慌）
└─ 综合建议：⚠️ 暂缓执行（黄金系受美元走势影响，非基本面恶化）
```

### 4.3 ETF 轮动信号挖掘

**新增能力**：
```
每周执行：
1. 扫描全市场 ETF（1917 只）
2. 挖掘轮动信号（哪些行业/主题即将启动）
3. 输出轮动建议：
   - 从 A 行业 ETF 轮动到 B 行业 ETF
   - 逻辑：估值差 + 动量切换 + 资金流
   - 置信度：高/中/低
```

## 六、风险与约束

### 6.1 数据约束
- **延迟容忍**：15 分钟级别数据，不适合高频信号
- **数据质量**：ETF 流动性/规模过滤（避免小 ETF 噪声）
- **历史长度**：至少 3 年数据，避免过拟合

### 6.2 模型约束
- **人在回路**：所有规则需人工审核确认
- **可解释性**：拒绝黑箱，每条规则必须有经济逻辑
- **复杂度上限**：规则≤5 条，避免过度拟合

### 6.3 执行约束
- **不提供买卖时点**：只输出分析框架和参考建议
- **风险提示**：所有建议附带风险说明
- **免责声明**：分析仅供参考，投资决策自行判断
