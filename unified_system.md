## 三、统一工作流程

### 3.1 日常监控（每日自动）

```
时间：交易日 16:00
任务：
  1. 读取 holdings_current.json（当前持仓）
  2. 获取最新行情（Tushare Pro）
  3. 计算持仓 ETF 实时盈亏
  4. 检查止损/止盈触发
  5. 输出：持仓简报 + 风控预警
```

### 3.2 定期调仓（每月自动）

```
时间：每月最后一个交易日 14:00
任务：
  1. 读取 holdings_current.json
  2. 运行 QuantaAlpha 策略引擎
  3. 生成目标持仓（Top 30 ETF）
  4. 对比当前持仓 vs 目标持仓
  5. 生成调仓指令（买入/卖出清单）
  6. 输出：调仓方案文档
```

### 3.3 触发调仓（事件驱动）

```
触发条件：
  - 单个 ETF 止损触发（-8%）
  - 单个 ETF 止盈触发（+20%）
  - 总止损触发（-10%）
  - 老板临时指令

任务：
  1. 读取 holdings_current.json
  2. 确认触发条件
  3. 生成智能分析（行业前景 + 仓位分析）
  4. 输出：执行建议（✅执行/🟡部分/⚠️暂缓/❌不建议）
```

## 七、关键联动机制

### 7.1 持仓更新联动
```
老板发送持仓截图
  ↓
OCR 识别（image-ocr skill）
  ↓
生成 holdings_current.json 临时文件
  ↓
发送老板确认
  ↓
确认后：
  - 更新 holdings_current.json
  - 创建备份（holdings_backup/）
  - 触发策略引擎重新计算
  - 更新每日简报
```

### 7.2 调仓执行联动
```
调仓日到达 / 触发器触发
  ↓
运行 unified_trading_engine.py
  ↓
读取 holdings_current.json（当前持仓）
  ↓
运行 QuantaAlpha 策略（生成目标持仓）
  ↓
对比当前 vs 目标 + 老板指令
  ↓
生成调仓指令（买入/卖出清单）
  ↓
发送飞书通知（老板审核）
  ↓
老板执行交易
  ↓
更新 holdings_current.json
```

### 7.3 风控联动
```
每日 16:00 监控
  ↓
检查持仓盈亏（holdings_current.json）
  ↓
触发条件判断:
  - 单个 ETF 止损（-8%）→ 飞书预警 + 智能分析
  - 单个 ETF 止盈（+20%）→ 飞书预警 + 智能分析
  - 总止损（-10%）→ 飞书警报 + 暂停策略建议
  ↓
老板决策 → 更新持仓
```
