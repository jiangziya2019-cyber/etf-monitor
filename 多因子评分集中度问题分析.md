## 二、根本原因分析

### 原因 1: 动量/波动因子数据缺失 ⚠️

**问题代码**:
```python
# multi_factor_v5.py
def calculate_momentum_factors(historical_data: Dict, date: str) -> Dict:
    if not historical_data or date not in historical_data:
        # 返回默认值 0.5
        return {code: {'score': 0.5} for code in etf_codes}
```

**现状**:
- 回测引擎调用时传入了 `historical_data={}`（空字典）
- 动量/波动因子函数检测到空数据，返回默认值 0.5
- **608 只 ETF 动量/波动评分全部相同！**

**影响**:
- 动量 (25%) + 波动 (20%) = **45% 权重失效**
- 实际有效权重仅 55%（估值/技术/资金/流动性/ETF 特有）

### 原因 2: Z-Score 标准化导致分数集中

**问题**:
```python
# Z-Score 标准化
z_scores = (values - mean) / std
z_scores = np.clip(z_scores, -3, 3)  # 限制在±3σ
scores = 0.5 + z_scores / 6  # 映射到 0-1

# 结果：大部分分数集中在 0.4-0.6 区间
```

**效果**:
- 极端值被压缩（Winsorization）
- 分数分布过于均匀
- 无法拉开差距

### 原因 3: 因子权重过于平均

**当前权重**:
```python
FACTOR_WEIGHTS = {
    'valuation': 0.30,    # 30%
    'momentum': 0.25,     # 25% (失效)
    'volatility': 0.20,   # 20% (失效)
    'technical': 0.10,    # 10%
    'fund_flow': 0.15,    # 15%
    'liquidity': 0.10,    # 10%
    'etf_specific': 0.10  # 10%
}
```

**问题**:
- 权重分散在 7 个因子
- 单一因子最大权重仅 30%
- 45% 权重失效后，区分度进一步降低

## 三、解决方案

### 方案 1: 修复动量/波动因子数据（优先级🔴）

**修复**:

**预期效果**:

- 波动因子：0.500 → 0.400-0.700（有区分度）
- Top 10 分差：0.097 → 0.150+（提升 50%+）

### 方案 2: 调整 Z-Score 参数（优先级🟡）

**当前**:
```python
z_scores = np.clip(z_scores, -3, 3)  # 限制±3σ
scores = 0.5 + z_scores / 6  # 映射到 0-1
```

**优化**:
```python
# 方案 A: 放宽限制
z_scores = np.clip(z_scores, -4, 4)  # 限制±4σ
scores = 0.5 + z_scores / 8  # 重新映射

# 方案 B: 使用百分位排名
def percentile_rank(series):
    """百分位排名（0-1）"""
    from scipy.stats import rankdata
    return rankdata(series) / len(series)

# 百分位排名能更好拉开差距
```

**预期效果**:
- 分数分布更分散
- Top 10 分差扩大 20-30%

### 方案 3: 增加核心因子权重（优先级🟡）

**当前**:
```python
'valuation': 0.30,  # 30%
'momentum': 0.25,   # 25%
```

**优化**:
```python
# 方案 A: 集中权重
FACTOR_WEIGHTS = {
    'valuation': 0.35,    # 提升到 35%
    'momentum': 0.30,     # 提升到 30%
    'volatility': 0.15,   # 降低到 15%
    'technical': 0.10,
    'fund_flow': 0.10,    # 降低到 10%
}

# 方案 B: 分层权重（推荐）
LAYER_WEIGHTS = {
    'defensive': {  # 第一层
        'valuation': 0.50,    # 50% 估值
        'volatility': 0.30,   # 30% 波动
        'liquidity': 0.20,    # 20% 流动性
    },
    'trading': {  # 第三层
        'momentum': 0.50,     # 50% 动量
        'technical': 0.30,    # 30% 技术
        'liquidity': 0.20,    # 20% 流动性
    }
}
```

**预期效果**:
- 核心因子主导评分
- 分层筛选更有针对性

### 方案 4: 引入非线性评分（优先级🟢）

**当前**: 线性加权
```python
composite = (
    0.30 * valuation +
    0.25 * momentum +
    ...
)
```

**优化**: 非线性增强
```python
# 方案 A: 平方增强（放大优秀因子）
composite = (
    0.30 * valuation**2 +
    0.25 * momentum**2 +
    ...
) / (0.30 + 0.25 + ...)  # 归一化

# 方案 B: 阈值过滤
if valuation < 0.4:  # 估值太差，直接淘汰
    composite = 0
elif momentum > 0.7:  # 动量极强，加分
    composite *= 1.2
```

**预期效果**:
- 优秀 ETF 得分更高
- 劣质 ETF 得分更低
- 分差扩大 30-50%
