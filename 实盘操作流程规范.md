## 三、特殊场景处理

### 3.1 触发器监控 (分阶段执行)

#### ⏸️ 第一阶段（第 1 周）：暂停所有触发器
**原因**:
- 仓位调整期，避免干扰正常调仓
- 目标 ETF 还没建仓完成
- 触发信号可能不准确

**替代方案**:
- ✅ 人工监控大额波动（>5%）
- ✅ 盘后统一调整仓位

#### 🟡 第二阶段（第 2 周）：仅开启止损触发
**监控时间**: 交易日 9:00-15:00，每 30 分钟
**监控脚本**: `etf_trigger_monitor_v2.py`
**阈值**: 跌幅>8%

#### ✅ 第三阶段（第 3 周起）：完整版触发器

#### 网格加仓触发 ⭐
**监控时间**: 交易日 9:00-15:00，每 15 分钟
**监控脚本**: `etf_trigger_monitor_v2.py`
**处理流程**:
1. 检测网格信号（价格跌破网格线）
2. 分析行业前景评分 (1-5 分)
3. 分析当前仓位 (该 ETF 占总仓位比例)
4. 给出执行建议：
   - ✅ 建议执行：行业前景≥4 分 + 仓位<5%
   - 🟡 部分执行：行业前景 3 分 + 仓位 5-8%
   - ⚠️ 暂缓执行：行业前景≤2 分 或 仓位>8%
   - ❌ 不建议执行：行业前景≤1 分
5. 发送飞书通知（App Bot API 私聊）

**通知模板**:
```
🔔 网格加仓触发 - XXXX

ETF: 510XXX
触发价：¥X.XXX
当前价：¥X.XXX
行业前景：X/5 分
当前仓位：X.X%
建议：✅ 建议执行 / 🟡 部分执行 / ⚠️ 暂缓执行 / ❌ 不建议执行
置信度：高/中/低
```

#### 止盈触发 ⭐
**监控时间**: 交易日 9:00-15:00，每 3 分钟
**监控脚本**: `etf_trigger_monitor_v2.py`
**处理流程**:
1. 检测止盈信号（涨幅>设定阈值）
2. 分析行业前景评分 (1-5 分)
3. 分析盈利情况 (浮盈比例)
4. 给出执行建议：
   - ✅ 建议执行：行业前景≤2 分 + 浮盈>10%
   - 🟡 部分止盈：行业前景 3 分 + 浮盈>15%
   - ⚠️ 暂缓止盈：行业前景≥4 分
   - ❌ 不建议止盈：行业前景 5 分 + 趋势良好
5. 发送飞书通知（App Bot API 私聊）

**通知模板**:
```
🔔 止盈触发 - XXXX

ETF: 510XXX
成本价：¥X.XXX

置信度：高/中/低
```

#### 止损触发 ⭐
**监控时间**: 交易日 9:00-15:00，每 3 分钟
**监控脚本**: `etf_trigger_monitor_v2.py`
