## 四、执行层面审查

### 4.1 建仓执行评估

#### ✅ 优势
- 分步执行（3 天完成）
- 灵活决策（周一 14:00 根据盘面）
- 整合现有系统

#### ⚠️ 潜在问题

| 问题 | 风险等级 | 说明 | 建议 |
|------|---------|------|------|
| **冲击成本未估算** | 🟡 中 | 大额买入可能推高价格 | 增加冲击成本估算 |
| **成交价格不确定** | 🟡 中 | 市价单可能成交价偏离 | 增加限价单策略 |
| **分批执行时机** | 🟢 低 | 分批时机依赖人工 | 增加智能分批 |

#### 📋 优化方案

**方案 A：冲击成本估算**
```
新增估算模型:

冲击成本 = (订单金额 / 日均成交) × 价格弹性系数

假设:
- 价格弹性系数：0.1（保守估计）
- 单笔订单<日均成交 10%

示例:
第三层建仓¥40,000，分 12 只 ETF
单只平均¥3,333，日均成交>5000 万
冲击成本 = (3333/50000000) × 0.1 = 0.0007%（可忽略）

结论:
- 当前资金规模冲击成本可忽略
- 如资金>100 万，需分批执行
```

**方案 B：限价单策略**
```
原方案：市价单

优化:
- 正常情况：市价单（快速成交）
- 波动>3%：限价单（±1% 范围）
- 流动性不足：限价单（±2% 范围）

执行:
- 系统自动判断市场状态
- 选择合适订单类型
- 未成交订单自动撤单/重下
```

**方案 C：智能分批执行**
```
原方案：固定 3 天分批

优化:
根据市场波动智能分批

- 低波动（<1%）：加速建仓（2 天完成）
- 中波动（1-3%）：按原计划（3 天完成）
- 高波动（>3%）：延缓建仓（5 天完成）

执行:
- 每日评估波动率
- 动态调整建仓节奏
- 发送调整通知
```
